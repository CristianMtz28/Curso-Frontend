Ramas

Las ramas son extensiones de nuestro proyecto original nuestro proyecto final y funcional debe, o sea la versión final debe estar en master; al momento de nosotros hacer un commit se crea una rama que es dónde trabajamos y git nos asigna un número de commit por ejemplo el d44e020. Estos nos funciona para tener una versión estable y funcional y modificar el código en las ramas de nuestro repositorio.

Para ver las ramas e que tenemos y en la que estamos trabajando usamos el comando:

git branch

Para poder crear una nueva barra usaremos el comando:

git checkout -b nombreDeLaRama

Con checkout podemos ir brincando entre las ramas

Para poder subir los cambios a master desde otra rama usamos el comnado

git merge nombreDeLaRama

Para borrar una rama debemos usar el comando

git branch -d nombreDeLaRama
