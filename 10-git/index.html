Repositorios

Los Repositoriosde Git son como se le conoce a los proyectos; para poder iniciar un repositorio debemos tener nuestra cuenta con github e iniciar con el comando:

git init

Este comando nos permite crear un nuevo repositorio vacio. Y si nosotros le damos en windows a "mostrar todos los archivos" veríamos una carpeta llamada .git que es toda la configuración

Para ver el status de nuestro repositorio usamos el comando:

git status

Cuando nosotros hagamos git status nos mostrará los commit que hayamos hecho previamente, los commits son las versiones que nosotros tenemos de nuestro proyecto; se recomienda hacer commits cada vez que nosotros movamos el código no importa que hagamos muchos commit eso es lo más normal

Ahora veremos el término del steign index. El steign index es una fase que nosotros tenemos antes de enviar nuestros archivos a una nueva versión. Esto se puede ver reflejado cuando ponemos gits status y nuestros archivos se ven en rojo; los que se hayan modificado, eso significa que aún no están en el staign index y si hacemos commit no revisará ni modificará ningún archivo; la misma terminal nos indica que debemos añadir (add) esos archivos. Para eso usamos el siguiente comando:

git add . = Con git add . mandamos al staign index a todos los archivos de nuestro repositorio

git add index.html Sin el punto y específicando el archivo que queremos enviar al staign index sólo asignamos un archivo

Después volvemos a poner git status

Lo cuál veremos nuestros archivos en verde y ya no en rojo, eso significa que nuestros archivos están en el staign index y están listos para crear una nueva versión de nuestro proyecto en nuestro repositorio

Después de eso pondremos el comando:

git commit -m "Mensaje de menos de 50 caracteres que sea descriptivo"

Este comando lo que hará es que guardará nuestro commit con el mensaje que específicamos con el -m y debe ser lo más descriptivo y acertado posible para saber que pasó la última vez que hicimos commit en nuestro repositorio

Pero que pasa si no queremos commitear un archivo de todos los que se subieron, supongamos que no queresmo commitear el styles.css entonces podemos usar el comando:

git rm --cached styles.css

Aquí usamos el rm --cached y el nombre del archivo que queremos remover del staign index.

Para corroborarlo usamos nuevamente el comando git status y pasará de verde a rojo

Por ultimo podemos y super recomendado podemos crear un archivo llamado:

.gitignore

En este archivo indicaremos que archivos no queremos que se vayan al staign index por default, por ejemplo archivos .pdf, .psd, .zip, o archivos concretos que nosotros querramos como no sé, un contatco.html etc para que a la hora de hacer git status no los prepare para ir al staign index
